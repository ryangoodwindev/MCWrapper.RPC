Class RpcOptions

RpcOptions values are mostly required for proper blockchain interaction to occur via the MCWrapper.RPC client.

The MCWrapper RPC client requires the following RpcOptions be assigned a valid value ChainUseSsl; ChainSslPath (ChainSslPath only required to be assigned a file path value if ChainUseSsl is true, otherwise ChainSslPath can be empty string ""); ChainUsername; ChainPassword; ChainName; ChainRpcPort; ChainAdminAddress; ChainHostname;

ChainSslPath

Local or remote file path to a copy of the same SSL certificate that is securing the current node the application is interacting with.

ChainSslPath is not required for the RPC client to function as expected;

Please note: If the ChainUseSsl value is true, the ChainSslPath value MUST be populated and accurate. When starting an HTTPS blockahin ensure you pass the -rpcssl flag and properly configure your multichain.conf file properly to support HTTPS connections.
